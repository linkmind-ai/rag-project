# 📊 검색 품질 평가 가이드 (Search Evaluation Guide)

이 문서는 `test_search_quality.py`를 활용하여 RAG 시스템의 검색 성능을 정량적으로 측정하고 최적화하는 방법을 안내합니다.

---

## 1. 🎯 평가의 목적
* **정량적 지표 수립**: "검색이 잘 된다"는 주관적 판단을 `Hit@K`, `MRR` 등의 수치로 객관화합니다.
* **최적의 가중치 도출**: 벡터 검색과 키워드 검색의 최적 조합(`vector_weight`)을 실험을 통해 결정합니다.
* **품질 회귀 방지**: 코드 수정 후 기존 검색 성능이 저하되지 않았는지 검증합니다.

---

## 2. 📚 데이터 구성: `golden_set.json`
`Golden Set`은 시스템이 반드시 찾아내야 할 **모범 답안지**입니다.

| 필드명 | 설명 | 비고 |
| :--- | :--- | :--- |
| `query` | 사용자의 실제 질문 또는 예상 질문 | |
| `expected_page_id` | 해당 질문에 대해 시스템이 찾아야 할 **정답 문서 ID** | |
| `keywords` | 검색 보조 키워드 (분석용) | |

---

## 3. 📈 핵심 지표 이해

### 3.1 Hit@K (정확도)
* **정의**: 상위 `K`개의 검색 결과 중 정답(`expected_page_id`)이 포함될 확률입니다.
* **해석**: `Hit@3 = 0.9`라면, 사용자가 첫 화면(3개)에서 답을 찾을 확률이 90%임을 의미합니다.

### 3.2 MRR (Mean Reciprocal Rank)
* **정의**: 정답의 순위 역수(`1/rank`)를 평균낸 값입니다.
* **계산 로직**: `1.0 / (idx + 1)` 수식을 사용하여 정답이 상위에 위치할수록 고득점을 부여합니다.
* **해석**: MRR이 1에 가까울수록 정답이 항상 1순위에 노출된다는 뜻입니다.

---

## 4. ⚖️ 하이브리드 검색 최적화 (`vector_weight`)
`hybrid_search`는 의미 기반의 **벡터 검색**과 정확한 단어 매칭인 **BM25**를 조합합니다.

* **`vector_weight = 0.5`**: 의미와 키워드의 균형을 맞춘 기본 전략입니다.
* **`vector_weight > 0.5`**: 질문의 의도가 추상적일 때 유리합니다.
* **`vector_weight < 0.5`**: 고유 명사나 특정 전문 용어가 포함된 질문에 유리합니다.
